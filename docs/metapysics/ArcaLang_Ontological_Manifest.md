§0.X. Политика тестовой валидации ArcaLang
0.X.1. Принцип дисциплинарной валидации

Тестовая система ArcaLang не является механизмом внешней проверки корректности.
Она служит для подтверждения:

    структурной целостности,

    операционной непрерывности,

    онтологической согласованности,

    функциональной полноты дистрибутива.

Тесты не определяют истину;
они фиксируют, что система остаётся работоспособной в рамках собственной онтологии.
0.X.2. Трёхзначная модель тестовой оценки

ArcaLang использует трёхзначную модель тестовой валидации:

    PASS — выражение принадлежит канону ArcaLang и интерпретируется корректно.

    INDEFINITE — выражение не принадлежит канону, но интерпретатор остаётся функциональным.

    FAIL — нарушение структурных инвариантов или операционный сбой.

INDEFINITE является допустимым и ожидаемым исходом для выражений,
которые не имеют онтологического статуса в ArcaLang.
0.X.3. Политика CLI

CLI ArcaLang является сервисным интерфейсом.
Он не обязан обеспечивать строгие exit‑коды классической UNIX‑модели.

Допустимы:

    exit-code = 0 — нормальное завершение;

    exit-code ≠ 0 — допустимо, если вывод содержит осмысленное сообщение
    (help, usage, описание ошибки, результат DSL).

Недопустимо:

    немое завершение без вывода;

    трассировка исключений.

0.X.4. Политика интерпретатора

Интерпретатор обязан:

    корректно обрабатывать канонические выражения;

    возвращать None для выражений вне канона;

    не поднимать исключений наружу;

    сохранять внутренние инварианты (иммониальность, трисмегистичность, непрерывность).

0.X.5. Каноническая трактовка неконструктивных выражений

Выражения, не являющиеся структурными элементами ArcaLang,
получают статус INDEFINITE.

Примеры:

    (0+2)^2

    (1+1)^2

    binary_transition(1,2)

    binary_transition(abc)

    пустые строки

    синтаксически неразборные конструкции

INDEFINITE не является ошибкой.
Это корректный результат в рамках дисциплины.
0.X.6. Цель тестовой политики

Цель тестовой политики ArcaLang —
обеспечить устойчивость дистрибутива при любой теоретической нагрузке,
включая выражения, не имеющие классической интерпретации.

Тесты подтверждают:

    непрерывность работы,

    отсутствие падений,

    сохранение онтологических инвариантов,

    функциональную полноту.

§0.5. Онтологические основания дисциплины
0.5.1. Иммониальный интерактив

Иммониальный интерактив — полнообусловленная предикативная числовая форма,
обладающая априорной числовой интерпретацией и существующая как рациональная связь двух агентов, параметров или объектов.

Свойства:

    не является суммой или операцией;

    существует как отношение, а не объект;

    обладает двойной статусностью:

        потенциальной,

        физической;

    всегда иммонирован (внутренне закреплён) в формализованных структурах ArcaLang.

0.5.2. Рациональная пара

Рациональная пара — минимальный структурный модуль ArcaLang.
Это иммониальный интерактив, обладающий:

    полнообусловленностью,

    предикативностью,

    априорной числовой природой,

    принадлежностью к непрерывному полю.

0.5.3. Аксиома рациональной пары

Пара в ArcaLang — это иммониальный интерактив,
обладающий априорной числовой интерпретацией
и порождающий трисмегистическую структуру.
0.5.4. Бинарная экстраполяция

Оператор:
E:(A,B)↦XABpot

переводит рациональную пару в потенциальную числовую форму.

Свойства:

    действует на предикативные формы;

    результат априорно числовой;

    допускает трисмегистическую фиксацию;

    является механизмом проявления структуры.

0.5.5. Трисмегист
T=(A,B,O)

где:

    A, B — рационально связанные параметры;

    O — наблюдатель / интегратор.

Трисмегист:

    фиксирует потенциальную форму;

    является минимальной аналитической структурой;

    является носителем Z‑морфологии.

0.5.6. Аксиома трисмегистической триады

Любая рациональная пара порождает трисмегистическую триаду.
0.5.7. Непрерывное поле

Непрерывное поле — онтологический носитель ArcaLang.

Свойства:

    отсутствие разрывов;

    отсутствие изолированных объектов;

    существование только отношений;

    декларативность координат;

    любая пара уже содержит третье.
