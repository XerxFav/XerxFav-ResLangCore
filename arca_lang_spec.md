# ArcaLang Specification

## 🧠 Онтологическая модель

ArcaLang — язык фазопространственного анализа, объединяющий логические, квантовые и философские структуры в исполняемую форму.

- **Фазы**: `neutral`, `positive`, `negative`
- **Переходы**: определены в `schema.py`
- **Интерпретация**: через `PhaseEvaluator`
- **Онтология**: генерируется из `headers.py` и `generate_ontology.py`

## 🧩 Модули

| Модуль              | Назначение                                 |
|---------------------|--------------------------------------------|
| `arca_core`         | Фазовая логика, хронотопы, заголовки       |
| `arca_fieldcore`    | Генерация онтологии, фазовые правила       |
| `arca_interpreter`  | Интерпретатор переходов                    |
| `arca_memory`       | Хранилище фаз                              |
| `tests`             | Проверка всех фазовых компонентов          |
| `docs`              | Спецификация и документация                |

## 🔁 CI-интеграция

- Тесты: `test_schema.py`, `test_evaluator.py`
- Покрытие: `htmlcov/`

Создать универсальный язык для моделирования, анализа и генерации фазовых структур в научных и философских системах.

# 🌌 Манифест Фундамента ArcaLang

## 🔹 Принцип
**«Относительное формирует и мотивирует абсолютное.»**

## 🔹 Основание
ArcaLang рождается из диалога бинарного и тернарного, из перехода от пустоты и единицы к мосту √2.  
Фундамент языка — это не просто арифметика, а философия перехода: от дискретного к непрерывному, от локального к универсальному.

## 🔹 Элементы
1. **OddPairOperator** — замещающий элемент, связывающий 0 и 1 через √2.  
2. **Аксиома бинарного перехода** — 

\[(0+1)^2 ≡ √2\]

, фиксирующая мост между относительным и абсолютным.  
3. **Мета‑правило согласования методов** — устранение конфликтности бинарного и тернарного анализа.  
4. **Диаграмма‑поток** — 0 → 1 → (0+1)² → √2 → непрерывность.  

## 🔹 Цель
ArcaLang утверждает согласованность аналитики, философии и кода.  
Каждый оператор — это шаг к целостности, каждый аксиоматический переход — это мост к непрерывному.

# 📐 Фундаментальные аксиомы

### Аксиома бинарного перехода
Формулировка:  


\[(0+1)^2 \equiv \sqrt{2}\]



Описание:  
Только конструктивная пара «0+1» формирует и мотивирует абсолютное.  
Её операторное возведение в квадрат даёт √2 — минимальную иррациональность, мост между дискретным и непрерывным.

Реализация:  
Модуль `arca_langlib/axioms/binary_transition.py` содержит класс `BinaryTransition`.  
Интерпретатор (`arca_interpreter/evaluator.py`) вызывает `BinaryTransition.apply()` при встрече выражения `(0+1)^2`.

## Оператор валидности и категориальная мода

ArcaLang вводит абстрактный оператор валидности для тройки параметров:

```arcalang
validity(P1, P2, P3) -> mode
let mode = validity(P1, P2, P3)


где:

    P1, P2, P3 — параметры, заданные в распределённо‑параметрическом объёме {4πR^3 r^2 (V - V0)},

    mode ∈ {even_category, odd_category}.

Интуитивно:

    even_category — валидность P3 согласована с валидностью пары (P1, P2),
    odd_category — P3 нарушает исходную валидность комбинации (P1, P2).
Минимальный пример использования:
let mode = validity(P1, P2, P3)
if mode == even_category {
    resonance::stable(P1, P2, P3)
} else {
    resonance::defect(P1, P2, P3)
}

Реализация по умолчанию отображается на ProtoFormOperator.classify_category(...).
